Hexo建立新文章後自動執行編輯器

2017-06-21

每次執行完 $ hexo new後都要去 \source\_posts裡面找出 .md檔，再用編輯器開啟真的是超麻煩的
這些功夫當然是能省則省

  1. 在 Hexo folder內建立新資料夾 scripts
  2. 在 scripts資料夾底下新增一個 .js檔案，檔案內容如下
    var exec = require(‘child_process’).exec;
    hexo.on(‘new’, function(data) {
    exec(‘start “C:\Users\XXX\AppData\Roaming\Haroo Studio\Haroopad\haroopad.exe“ ‘+ data.path);
    });
  3. 在 Git Bash執行 $ hexo new指令看看

最後找出錯誤是因為我在程式碼最後 }); 又隔一行然後放了註解 …

從零開始使用 Github Page建立 Hexo個人部落格

2017-06-21

replace red text by yourself

  1. install node.js
  2. install git
  3. create new folder Hexo folder
  4. enter Hexo folder
  5. right click and select Git Bash Here
  6. enter command line: $ npm install -g hexo-cli
  7. enter command line: $ hexo init
  8. //enter command line: $ npm install
  9. enter command line: $ hexo server
    (go to local page: http://localhost:4000/)
  10. enter command line: $ hexo generate (generate at folder “public”)
  11. modify deploy setting at _config.yml
    deploy:
    type: git
    repository: git@github.com:Mortiz09/mortiz09.github.io.git
    branch: master
  12. enter command line: $ npm install hexo-deployer-git –save
  13. enter command line: $ git config –global user.name “mortiz09“
  14. enter command line: $ git config –global user.email “xxxxx@hotmail.com“
  15. enter command line: $ ssh-keygen -t rsa -C “xxxxx@hotmail.com“
  16. open C:\Users\Moz\.ssh\id_rsa.pub, copy all text
  17. go to github, Settings, SSH and GPG keys
  18. click “New SSH key”, paste to input , click “Add SSH key”
    (13-18 ref: http://blog.csdn.net/jingtingfengguo/article/details/51892864)
  19. enter command line: $ hexo deploy

Finally!! Go to your page: https://mortiz09.github.io/
